# Глоссарий — Единый язык (Ubiquitous Language)

Этот документ — единый источник истины для терминологии проекта.  
Все участники (разработчики, менеджеры, тестировщики) используют одни и те же названия в коде, БД, документации и в разговоре.

---

## Основные сущности

| Русский термин | English term | Техническое название | Описание |
|----------------|-------------|----------------------|----------|
| Организация | Organization | `Organization` | Спортивная организация-клиент системы. Все данные изолированы по `org_id`. |
| Клиент | Client | `Client` | Посетитель/участник организации. |
| Абонемент | Membership | `Membership` | Купленный клиентом абонемент. Имеет баланс занятий и срок действия. |
| План абонемента | Membership Plan | `MembershipPlan` | Шаблон/тариф, на основе которого продаётся абонемент. Определяет цену, период и кол-во занятий. |
| Занятие | Session | `Session` | Конкретное проведённое или запланированное занятие группы (ранее `GroupInstance`). |
| Группа | Group | `Group` | Шаблон повторяющегося курса/класса с расписанием. Например: «Йога ПН/СР». |
| Слот расписания | Schedule Slot | `ScheduleSlot` | Один временной слот в расписании группы (день недели + время начала/конца). |
| Посещение | Attendance | `Attendance` | Факт посещения клиентом занятия. Проходит через статусы: REGISTERED → ATTENDED / CANCELLED. |
| Зал | Room | `Room` | Физическое помещение, в котором проводится занятие. |
| Сотрудник / Тренер | Employee / Trainer | `Employee` | Сотрудник организации. Роль `TRAINER` — тренер, ведущий занятия. |
| Дисциплина | Discipline | `Discipline` | Вид спорта или направление занятий (йога, бокс, плавание и т.д.). |
| Платёж | Payment | `Payment` | Регистрация факта оплаты абонемента. |

## Атрибуты и поля

| Русский термин | English term | Техническое название | Описание |
|----------------|-------------|----------------------|----------|
| Баланс занятий | Session balance | `Membership.balance` | Количество оставшихся занятий в абонементе. |
| Срок действия | Expiry date | `Membership.expires_at` | Дата истечения абонемента. |
| Кол-во занятий в плане | Sessions count | `MembershipPlan.sessions_count` | Сколько занятий даёт план при покупке. |
| Период действия (дней) | Period (days) | `MembershipPlan.period_days` | Длительность плана в днях. |

## Типы планов абонементов

| Русский термин | English term | Код (`MembershipPlanType`) | Описание |
|----------------|-------------|---------------------------|----------|
| Классический | Classic | `CLASSIC` | X занятий за период (например, 12 занятий за 30 дней). |
| Безлимитный | Unlimited | `UNLIMITED` | Неограниченное количество занятий за период. |
| День-в-день | Drop-in | `DROPIN` | Разовое посещение, без накопления. |
| Пакет занятий | Package | `PACKAGE` | X занятий без срока (не привязано к календарю). |

## Статусы

### Membership (абонемент)

| Статус | Описание |
|--------|----------|
| `PENDING_PAYMENT` | Продан, ожидает оплаты. |
| `ACTIVE` | Оплачен, действующий. |
| `EXPIRED` | Истёк срок (`expires_at < now`). |
| `EXHAUSTED` | Кончился баланс занятий (`balance == 0`). |
| `CANCELLED` | Отменён вручную. |

### Session (занятие)

| Статус | Описание |
|--------|----------|
| `SCHEDULED` | Запланировано, ещё не проведено. |
| `COMPLETED` | Проведено тренером (`CompleteSession`). Именно в этот момент списываются занятия. |
| `CANCELLED` | Отменено (болезнь тренера и т.п.). Все посещения → `CANCELLED` без списания. |

### Attendance (посещение)

| Статус | Описание |
|--------|----------|
| `REGISTERED` | Черновик: клиент отмечен тренером, занятие ещё не завершено. Баланс **не** списывается. |
| `ATTENDED` | Финал: занятие завершено, баланс списан. |
| `NO_SHOW` | Не пришёл. |
| `CANCELLED` | Посещение отменено (с восстановлением баланса, если было `ATTENDED`). |

## Ключевые операции (Use Cases)

| Операция | Техническое название | Описание |
|----------|---------------------|----------|
| Создать план абонемента | `CreateMembershipPlan` | Администратор создаёт шаблон для продажи. |
| Продать абонемент | `CreateMembership` | Продажа абонемента клиенту на основе плана. |
| Отметить посещение | `CheckinClient` | Тренер регистрирует клиента → `REGISTERED` (баланс не меняется). |
| Завершить занятие | `CompleteSession` | Тренер завершает занятие → все `REGISTERED` → `ATTENDED`, баланс списывается атомарно. |
| Отменить посещение (черновик) | `RemoveAttendance` | До завершения занятия, без эффектов на баланс. |
| Отменить посещение (финал) | `CancelAttendance` | После завершения, восстанавливает баланс. |
| Списать занятие | `DeductMembership` | Внутренняя операция: `Membership.balance -= 1`. Происходит в `CompleteSession`. |
| Восстановить занятие | `RestoreMembership` | Внутренняя операция: `Membership.balance += 1`. Происходит в `CancelAttendance`. |

## Правила именования в коде

- **Классы:** `PascalCase` — `MembershipPlan`, `Session`, `Attendance`
- **Таблицы:** `snake_case` — `membership_plans`, `sessions`, `attendance`
- **API endpoints:** `kebab-case` — `/api/membership-plans`, `/api/sessions`
- **Error codes:** `SCREAMING_SNAKE_CASE` с namespace — `PLAN_NOT_FOUND`, `MEMBERSHIP_EXPIRED`, `SESSION_ALREADY_COMPLETED`
- **Enum values:** `SCREAMING_SNAKE_CASE` — `MembershipPlanType.CLASSIC`, `SessionStatus.COMPLETED`
